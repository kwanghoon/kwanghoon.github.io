How to Install and Run

A Few Preliminaries

  • Java 1.8 JRE (SE), Android Studio 1.2.2 (including SDK)

  • Separators in Java class path

    • On Windows, use a semicolon as -cp "bin;lib/*;tool/*".

  • Make sure that the binaries (GenTestsfromIntentSpec/bin/{gen_mac,gen.exe}) are executable.

    • (Particularly, on Mac, you can check "r-x r-x r-x" permission of gen_mac by ls -l AndroidAppTester/GenTestsfromIntentSpec/bin/gen_mac.)

GUI-based Interactive Mode for a Single App

  • Download the up-to-date binary package (AndroidAppTester.zip) from the software site into, for example, D:\ on Windows. Then,

    • D:\> unzip AndroidAppTester.zip
    • D:\> cd AndroidAppTester\UIforIntentSpec
    • D:\AndroidAppTester\UIforIntentSpec> java -cp "bin;lib/*;tool/*" kr_ac_yonsei_mobilesw_UI.Main
      • (Do run the java command in ${Unzipped Directory}\UIforIntentSpec})
      • (On Mac, the class path must be seprated by a colon, not by a semicolon as -cp "bin:lib/*:tool/*")

  • Generating ADB commands from an intent specification {cmp=Activity com.example.android/.MainActivity}, which is written, for example, by a programmer.

    • Step 1 : You write the intent specification in the text window. Then press "ADB Command" button.
    • Step 2 : Press "Generate ADB Command" button to see ADB commands generated in the bottom of the screen.

  • Run the ADB commands generated from a saved intent specification file, which is generated from AndroidManfiest.xml in the example.

    • Step 1 : Instead of writing an intent specification, you can import it from an AndroidManifest.xml. Press "Import" button to find it in D:\AndroidAppTester\Example\NotesList\app\src\main.
    • Step 2 : You can see an imported intent specification from the AndroidManifest.xml. Then press "ADB Command" button.
    • Step 3 : Press "Generate ADB Command" button to generate a set of ADB commands. Then press "Run" button.
    • Step 4, Step 5, Step 6 : Press "Find ADB" button to locate ADB.exe (on Windows). You can see an ADB path properly set up in the step 6.
      • Before you do Step 4, 5, and 6, either an Android device should be connected to your PC via USB or you should launch an Android emulator.
      • Also, you need to build NotesList app and to install on the target Android device or emulator. You can use Android studio to do these.
    • Step 7 : Press "ReadDevice" button to find the ID of Android Device to which we send the generated ADB commands to.
    • Step 8 : Press "ViewLogcat" button to receive logs from the connected Android device.
    • Step 9, Step 10 : Press "Run" button to start a testing by sending ADB commands to the Android device. You see a screen shot of the Android device under testing in Step 10.
    • Step 11 : You have finished testing all ADB commands, and you can see the test results (Pass/Fail).
    • Step 12 : Later, you can review all the logs recorded in Microsoft Excel file.

  • Generating Android JUnit Test class files from a saved intent specification file, which is generated from AndroidManfiest.xml in the example.

    • Refer to the Screen shot site for this feature.

Batch Mode for Multiple Apps

  • Download the up-to-date binary package (AndroidAppTester.zip) from the software site into, for example, D:\ on Windows. Then,

    • D:\> unzip AndroidAppTester.zip
    • D:\> cd AndroidAppTester\UIforIntentSpec
    • D:\AndroidAppTester\UIforIntentSpec> mkdir apks
    • D:\AndroidAppTester\UIforIntentSpec> copy ..\Example\NotesList\app\build\outputs\apk\app-debug.apk apks\
    • D:\AndroidAppTester\UIforIntentSpec> start.bat apks
      • We recommed to connect to an Android mobile phone for this batch mode.
